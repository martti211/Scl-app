# SCL — Student Conduct Log

## Deployment auf GitHub Pages (iPhone/iPad)

-----

### Schritt 1: GitHub-Account

Falls noch kein Account: https://github.com → Sign up (kostenlos, 2 Minuten)

-----

### Schritt 2: Neues Repository anlegen

1. github.com → grüner Button **“New”**
1. Repository name: `scl-app` (genau so)
1. **Public** lassen
1. **“Create repository”** klicken

-----

### Schritt 3: Dateien hochladen

Im neuen Repository auf **“uploading an existing file”** klicken.
Diese 5 Dateien hochladen:

- `index.html`
- `manifest.json`
- `sw.js`
- `icon-192.svg`
- `icon-512.svg`

Dann: **“Commit changes”** → grüner Button unten.

-----

### Schritt 4: GitHub Pages aktivieren

1. Im Repository → **Settings** (oben rechts)
1. Linke Spalte → **Pages**
1. Source: **Deploy from a branch**
1. Branch: **main** → Ordner: **/ (root)**
1. **Save** klicken

Nach 1-2 Minuten erscheint die URL:
`https://DEIN-USERNAME.github.io/scl-app/`

-----

### Schritt 5: Als App auf iPhone/iPad installieren

1. Diese URL in **Safari** öffnen (nicht Chrome!)
1. Teilen-Button (Quadrat mit Pfeil nach oben) tippen
1. **“Zum Home-Bildschirm”** wählen
1. Name lassen oder anpassen → **Hinzufügen**

Die App erscheint jetzt mit eigenem Icon auf dem Homescreen.
Öffnet sich wie eine native App — kein Browser-Chrome, Vollbild.

**Auf iPad identisch** — dieselbe URL, dasselbe Vorgehen.

-----

### Schritt 6: Google Sheets verbinden (optional, für direkten Export)

1. `SCL_Vorlage.xlsx` in Google Drive hochladen → als Sheets öffnen
1. Erweiterungen → Apps Script → `SCL_AppsScript.js` einfügen
1. Zeile `DEINE_SPREADSHEET_ID_HIER` ersetzen (ID aus der Sheets-URL)
1. Bereitstellen → Als Web-App → URL kopieren
1. In der SCL-App: Kürzel-Pill (LK) lang drücken → URL eintragen

Ab dann schreibt die App direkt ins Sheet — kein manueller Schritt mehr.

-----

### Offline-Funktion

Die App funktioniert nach dem ersten Laden auch ohne WLAN:

- Klassen und Namenslisten: immer verfügbar (lokal gespeichert)
- Erfassung und Speicherung: immer verfügbar
- KI-Parsing (Sprache → Einträge): braucht kurz Internet
- Dossier aus lokalem Speicher: immer verfügbar

-----

### Updates einspielen

Wenn eine neue Version der App da ist:

1. `index.html` in GitHub ersetzen (hochladen, alte Datei löschen)
1. In der App: Seite neu laden — fertig
